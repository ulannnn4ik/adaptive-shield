Ключевые преимущества
Adaptive Shield работает на уровне L7 (приложения) — анализирует содержимое запросов, поведение пользователей и паттерны сессий
fail2ban реактивен (анализирует логи постфактум) — Adaptive Shield проактивен (анализирует в реальном времени до обработки запроса)
nginx rate limiting не сохраняет состояние — Adaptive Shield поддерживает репутацию IP между сессиями с затуханием
Тек Adaptive Shield обнаруживает credential stuffing и отличает его от целенаправленного brute force

Методология

Тесты проводились в локальной среде. Каждая атака 30 секунд симулировалась. Базовый трафик: 5 запросов/сек. Метрики собирались через потоки событий Redis.

Процент блокировки = заблокированные вредоносные запросы / все вредоносные запросы × 100%
False Positive Rate = заблокированные легитимные запросы / все легитимные запросы × 100%
Время обнаружения = время от первого запроса атаки до первой блокировки

Why Admin Panels Are Targeted
DDoS Against Admin
Flooding the admin endpoint with thousands of requests per second to crash the management interface and blind the defender.
Brute Force on Admin Login
Automated tools like Hydra try thousands of admin:password combinations в минуту until access is granted.
Credential Stuffing
Using leaked database dumps to test admin credentials. One match gives full control of the server.
Adaptive Shield Response
Rate limit + lockout + IP ban + Telegram alert — all triggered within milliсекунд of attack detection.

DDoS Attack Types
HTTP Flood (Layer 7)
Sends thousands of valid HTTP GET/POST requests. Hard to block — looks like real traffic. Adaptive Shield detects via EMA baseline deviation.
SYN Flood (Layer 4)
Sends millions of TCP SYN packets without completing handshake. Blocked at network layer by iptables rules.
Slowloris (Layer 7)
Holds connections open with slow headers. Nginx timeout config kills these within 10 секунд.
UDP Flood (Layer 3)
Floods with UDP packets. Blocked at iptables level before reaching the application.
