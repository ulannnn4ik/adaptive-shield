Как это работает
От запроса до ответа
01
Запрос поступает
Клиент отправляет HTTP запрос. Middleware Shield перехватывает его до того, как он достигнет приложения.
02
Проверка репутации IP
IP проверяется в базе репутации. Забаненные IP получают 403 мгновенно.
03
Оценка rate limit
Token Bucket и Sliding Window проверяют текущую частоту запросов против адаптивных лимитов.
04
Оценка аномалий
Z-score отклонение от EMA baseline. Значение выше 2.5σ — потенциальный DDoS.
05
Разрешить или заблокировать
Чистые запросы проходят. Угрозы получают 429 или 403. Все события логируются в SQLite.

Exponential Moving Average
Адаптивный базовый уровень
EMA запоминает нормальный уровень трафика и автоматически адаптируется. В отличие от статичного порога — ночью лимит ниже, в час пик выше. Когда трафик резко превышает baseline × burst multiplier — лимиты ужесточаются.
~50ms
Время реакции
96%
DDoS блокировка
0.1
EMA α (smoothing)
# EMA обновление
baseline = α × current_rps + (1-α) × baseline
limit = baseline × burst_multiplier
# Если rps > limit → блокировка
⚖️
Сравнение с аналогами
Почему EMA лучше
Статичный rate limiting (nginx, fail2ban) использует фиксированный порог — он либо слишком строгий (блокирует легитимных), либо слишком мягкий (пропускает атаки). EMA балансирует автоматически.
Статичный порог Не адаптивный ✗
Sliding Window Частично ≈
EMA (Adaptive Shield) Полностью адаптивный ✓

Z-score Anomaly Detection
Статистическое отклонение
Z-score измеряет насколько текущий трафик отклоняется от нормы в единицах стандартного отклонения. При z > 2.5σ система повышает репутационный штраф для IP и включает режим повышенной бдительности.
2.5σ
Порог аномалии
30
Обнаружено
0ms
Overhead
# Z-score вычисление
z = (x - mean) / std_dev
# Если z > 2.5 → аномалия
if z > THRESHOLD: add_violation(ip)

Distributed DDoS защита
Глобальный счётчик трафика
Когда атака идёт с тысяч разных IP — каждый по отдельности выглядит нормально. Глобальный счётчик суммирует весь трафик. При превышении 500 req/10сек — режим тревоги. При 1000+ — временная блокировка всех новых IP.

Прогрессивная блокировка

Credential Stuffing Detection
Обнаружение подбора аккаунтов
Credential stuffing — атака где один IP пробует войти в разные аккаунты. Если с одного IP приходит 3+ разных username за короткий период — это сигнал атаки. IP получает дополнительный репутационный штраф.
